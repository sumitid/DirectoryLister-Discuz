主题特点 基于Directory Lister二次开发，融合了经典的Discuz!4.1.0论坛风格，打造怀旧感十足的目录列表程序。保留原程序的简洁性，同时加入个人审美偏好。 - **Discuz!4.1.0经典界面样式** - 蓝色渐变导航栏、白色主内容区、黄色公告栏 - **支持中文目录和文件名** - 完美显示中文 - **支持文件夹内README说明** - 每个文件夹可放置README.html显示简介 - **横向文件列表** - 文件名 | 文件大小 | 最后修改时间 三列布局 - **友情链接框** - 45px高度，可自由添加友链 - **独立页脚文件** - 便于修改版权信息 - **所有CDN文件本地化** - 无需外部依赖 - **完全响应式** - 手机端自动适配 --- ### 更新记录 - 2026/02/02，发布Discuz!4.1.0经典主题，完全重构界面样式 - 2026/02/09，优化文件列表为横向布局 - 2026/02/12，增加友情链接 - 2026/03/03，友情链接改成独立文件 - 2026/03/03，间距调整为5px，布局更紧凑 --- ### 程序放在网站子目录不显示README.html 的解决方法 因为程序有个判断 `README.html` 路径的代码，而如果是正常使用域名或IP，都是可以自适应的。 但是如果把程序放在子目录下，就会无法获取正确 `README.html` 路径，需要你手动修改下程序里的每一行代码。 假设你将程序放在了子目录 `discuz41` 中（也就是 `http://xxx.xx/discuz41`）才能访问到程序网页。 首先打开该文件： `/index.php` 找到第5行的： `$suffix_array = explode('.', $_SERVER['HTTP_HOST']);` 将其修改为： `$suffix_array = explode('.', $_SERVER['HTTP_HOST'].'/discuz41');` --- ### 简介功能说明 可以在每个文件夹下面放一个 `README.html` 文件，这个文件里写着简介说明内容即可。 为了避免中文乱码，把 `README.html` 文件用 UTF-8 无BOM编码保存！ **注意**：本主题已将README显示位置调整到网盘框下方、友情链接上方，间距5px。 --- 项目根目录/ ├── index.php # 主文件 ├── header.php # 头部文件 ├── footer.php # 页脚文件 ├── links.php # 友情链接文件 ├── README.md # 项目说明 └── resources/ └── themes/ └── Discuz41/
